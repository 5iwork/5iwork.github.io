License控制

License 分在线和离线两种，在线一般为授权集中控制，要求客户端需要能连License服务器。离线一般为技术加业务双重控制，适用于部署环境的网络隔离。本文主要针对离线控制做相应的分析和描述。

1. 技术控制

由于java字节码基本等同于源码，所以很难在技术上绝对控制，只能在一定程度上增加破解的难度。下面是几个增加难度的点：

  1. 设置多个加密点

  2. 设置多种触发方式

    1. 使用触发，在涉及控制的业务使用时触发，结合业务

    2. 延时触发，触发时，先不要做出保护动作，让软件正常运行，这样就很难一次性把所有加密点全都找到，能有效增加一次性破解的难度

  3. 设置多检测机制

    1. 启动检测，每次启动时监测，被破解或者超过限制无法启动

    2. 定时检测，比如一周监测一次

    3. 特定条件检查，在满足特定条件是检测

    4. 随机检测，在业务使用过程中随机检测

  4. 保护动作

    1. 如果探测到非法使用行为，可以将软件锁定，比如无法启动或者拒绝提供服务

    2. 如果探测到破解，可以暂时加密关键数据，保护数据

2. 业务控制

  1. 在软件授权协议中约定，如果有调试、破解、反编译等行为，需要支付违约金
